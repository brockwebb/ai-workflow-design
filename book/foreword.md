## On AI-Assisted Authoring

This book was designed and authored using the same design discipline it teaches. The intellectual architecture, the chapter structure, the argument flow, the editorial decisions, the audience framing, and all judgment calls are the author's. AI tools did execution work: drafting prose from detailed specifications, reformatting content, research assistance, citation verification, and build system automation.

Every chapter began as a detailed specification document written by the author, defining the thesis, argument flow, source material, tone, and acceptance criteria. The AI partner executed against those specifications. The resulting drafts were reviewed, revised, and accepted (or rejected and rewritten) by the author. This is the same specification-before-execution pattern described in Chapter 10, applied recursively to the book's own production.

The recursive application is worth noting: the methodology this book teaches was used to write the book. The design patterns, the configuration control discipline, the provenance tracking, the iterative refinement with external validation, all of it was operational throughout the authoring process. Where the methodology worked well, the book reflects it. Where it created friction or failed, those experiences informed the content.
