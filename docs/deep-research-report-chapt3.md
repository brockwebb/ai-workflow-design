# Research Examples by Chapter

## Chapter 3: LLM-assisted data cleaning and imputation  
- **Tian, W., Lu, Y., Ding, Y., & West, B. T. (2025).** *Comparing Large Language Models and Traditional Methods for Imputing Missing Survey Responses in a 2024 U.S. Presidential Election Survey* (AAPOR 2025 conference). This study used real survey data from a 2024 U.S. election study with induced nonresponse. They compared LLM imputation (Llama 2 13B with few-shot prompts) to traditional methods (MICE, KNN). The LLM achieved 0.65–0.66 accuracy (F1) on imputing categorical responses, exceeding MICE’s 0.50–0.58 and KNN’s 0.29–0.55【35†L545-L546】. (Real survey data; not a pure NSO, but direct comparison with standard imputation methods; LLM ran ~40× slower than MICE【33†L286-L294】.)  
- *No published government report was found describing LLMs in official stats data cleaning*, but academic work suggests LLMs can complement traditional imputations.  Tian et al. is a concrete example using real survey data and shows LLMs may improve on some variables while noting computational cost【35†L545-L546】. (No direct U.S. agency case was cited; this is published research on survey data.)  

## Chapter 4: LLM-assisted entity extraction and disclosure review  
- **DiGiuseppe, M. R., & Flynn, M. E. (2026).** *Scaling Open-Ended Survey Responses Using LLM-Paired Comparisons*. *Public Opinion Quarterly*, nfag013. This peer-reviewed study used a real open-ended question on interest-rate knowledge and compared LLM-based pairwise comparisons to naive 0–10 ratings. The LLM-driven Bradley–Terry approach yielded more consistent, fine-grained scales, aligning closely with expert coding. (Real survey responses; academic social science research.) The authors found that using six different LLMs for pairwise comparisons gave more reliable scores than zero-shot numeric ratings【40†L1-L4】. (Not from an official agency, but directly relevant to coding open-ended survey data.)  
- **Templ, M. (2026).** *AI-assisted statistical disclosure control with sdcMicro* [R package vignette]. R Foundation. This vignette describes software (sdcMicro) integrating LLMs for disclosure avoidance. It provides `AI_createSdcObj()` to classify variables as quasi-identifiers and `AI_applyAnonymization()` to propose anonymization strategies. The LLM only sees metadata (variable names, types, categories) and returns human-readable reasoning and reproducible R code【42†L47-L55】. (Demonstration with real microdata examples; Austrian Statistical Office context—Templ is a known official stats researcher.) This is a practical tool for statistical offices, showing LLMs aiding disclosure review without sending sensitive data【42†L47-L55】.  

## Chapter 5: Multi-model/ensemble LLM pipelines with evaluation  
- **Fagerberg, P., Sallander, O., Vikhe Patil, K., et al. (2025).** *Dual-Model LLM Ensemble via Web Chat Interfaces Reaches Near-Perfect Sensitivity for Systematic-Review Screening: A Multi-Domain Validation with Equivalence to API Access*. *Preprint* (medRxiv). In medical literature screening, a dual-LLM ensemble (OpenAI GPT-5 “Thinking” and Google Gemini 2.5) with an “OR” rule attained 99.7% sensitivity on Cochrane abstracts (vs ~94–98% for the best single models), trading off specificity (~49%)【56†L100-L108】. (Retrospective real data from 16 Cochrane reviews.) This shows that combining two model families can “mitigate model-specific blind spots” and achieve near-perfect recall, equivalent to API-based runs【56†L100-L108】. (Not an official stats case, but exemplifies a multi-LLM ensemble with quantitative evaluation.)  
- **Liu, X., et al. (2025).** *SE-Jury: An LLM-as-Ensemble-Judge Metric for Narrowing the Gap with Human Evaluation in Software Engineering*. *Preprint* (ArXiv). This study builds an ensemble of LLM evaluators (e.g. equivalence-checking, test-case-based) to score code generation tasks. On several benchmarks (code generation and summarization), the SE-Jury ensemble (with dynamic team selection) achieved 30–140% higher Kendall/Spearman correlation with human judgments than single-model metrics【60†L31-L40】. (Real annotated datasets for coding tasks.) Thus multiple LLM evaluators combined yielded reliability close to human; a form of “LLM-as-judge” ensemble with measured improvements【60†L31-L40】. (Software engineering focus, but methodology parallels the use of judges/critics for LLM outputs.)  
- **Kamen, A., & Kamen, Y. (2025).** *Majority Rules: LLM Ensemble is a Winning Approach for Content Categorization*. *Preprint* (ArXiv). Using ten state-of-the-art LLMs zero-shot, they form ensembles (eLLM) for hierarchical taxonomy classification (IAB 8,660-sample corpus). Ensembling with majority vote improved F1 by up to 65% over the best single model, achieving near expert-level accuracy【69†L52-L60】. (Real labeled data.) The ensemble reduced hallucinations and inconsistency seen in individual LLMs and “approaches or exceeds consistency of human expert annotation”【69†L52-L60】. (Though not official-stats data, this architecture clearly shows quantitative gains from multi-LLM pipelines.)  

## Chapter 8: Evaluation frameworks for LLM pipelines in high-stakes domains  
- **Tam, T. Y. C., Sivarajkumar, S., Kapoor, S., et al. (2024).** *A framework for human evaluation of large language models in healthcare derived from literature review*. *NPJ Digital Medicine*, 7, 258. This systematic review (2018–Feb 2024) of LLM evaluations in healthcare proposes “QUEST”: five evaluation dimensions (Quality of information, Understanding/Reasoning, Expression/Persona, Safety/Harm, Trust/Confidence). The framework provides standard criteria and example metrics/questions for human evaluation of LLM outputs in clinical settings【84†L75-L81】. (The work is based on peer-reviewed studies; the framework itself is conceptual.) It addresses the need for consistent, structured human eval in regulated domains like healthcare, which parallels official stats in requiring accuracy and transparency【84†L75-L81】.  
- **Azanza, M., Pérez Lamancha, B., & Pizarro, E. (2025).** *Tracking the Moving Target: A Framework for Continuous Evaluation of LLM Test Generation in Industry*. *In Proc. Evaluation and Assessment in Software Engineering (EASE 2025)*. They propose a practical CI/CD pipeline for continuously evaluating LLM-powered test generators. The framework integrates with standard tools (e.g. SonarQube), applies continuous regression tests, and tracks metrics like test coverage and maintainability. In a longitudinal industry study, it flagged when LLM performance drifted and ensured reproducibility【90†L63-L71】. (Applied to real industrial projects.) Though focused on software, it illustrates continuous evaluation: automated golden outputs, drift monitoring, human-in-the-loop validation, etc. for high-stakes use.  
- **Chen, J., Xu, X., Wei, H., Chen, C., & Zhao, B. (2026).** *SWE-CI: Evaluating Agent Capabilities in Maintaining Codebases via Continuous Integration*. *Preprint* (ArXiv). This work introduces SWE‑CI, a benchmark of 100 real-world software evolution tasks (avg. 71 commits over 233 days). It shifts evaluation from one-shot fixes to long-term maintainability: LLM agents must handle dozens of iterations of changing requirements. This “CI loop” benchmark measures how well agents sustain code quality over time【88†L61-L69】. (Real code repository histories.) It exemplifies a continuous evaluation setup with regression tracking, relevant to any high-stakes pipeline needing stability and reproducibility over time【88†L61-L69】.  

**Sources:** Each citation above is peer-reviewed or official (or peer-endorsed preprint) work as noted. The summaries state whether real data or benchmarks were used, and any connection to official statistical production or analogous workflows. Vendors and non-peer materials were excluded.
