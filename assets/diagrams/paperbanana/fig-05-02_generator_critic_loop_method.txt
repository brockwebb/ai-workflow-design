# fig-05-02: Generator-Critic Loop Topology

## Diagram Type
Technical flow diagram with a feedback loop. Clean, minimal, professional. Matches series style.

## Layout
Left-to-right primary flow with a downward feedback arc from Critic back to Generator. Four termination condition branches exit the loop to the right, converging on a single output. The loop structure is the visual centerpiece.

## Elements

### Input
1. Rectangle on far left: "Input"
   Small text below: "Raw source — administrative record, form, unstructured text"
   Arrow to Generator box.

### Stage 1: Generator
2. Box: "Generator (Model A)"
   Small text: "Produces structured output against target schema"
   Arrow from Input into this box.
   Arrow continuing right to Critic box (main loop flow).

### Stage 2: Critic
3. Box: "Critic (Model B)"
   Small text: "Evaluates output: schema validity, semantic coherence, completeness"
   Arrow from Generator into this box.

### Stage 3: Evaluation Decision (four exits)
4. Diamond: "Passes?"
   Arrow from Critic into this diamond.
   Four exits from this diamond, each labeled:
   - "All checks pass" → Output (right)
   - "Max iterations reached" → Output with flag (right)
   - "Confidence threshold met" → Output (right)
   - "Unresolvable" → Human Escalation (right)

### Feedback Loop
5. "Fail — with reasons" label on downward arrow from diamond.
6. Box below main flow: "Rejection + Rationale"
   Small text: "Specific, actionable — not 'try again'"
   Arrow from Fail branch into this box.
   Arrow looping back left and up to Generator (feedback arc).
   Small annotation on feedback arc: "Iteration counter increments"

### Termination Outputs
7. Box: "Human Escalation Queue"
   Small text: "Unresolvable by critic — requires domain expert"
   Arrow from "Unresolvable" exit into this box.

8. Box: "Flagged Output"
   Small text: "Max iterations reached — pass with quality flag"
   Arrow from "Max iterations" exit into this box.

### Stage 4: Provenance
9. Box: "Provenance Record"
   Small text: "Generator version · Critic version · Iteration count · Rejection reasons"
   All termination paths (pass, flagged, escalated) feed into this box.

### Output
10. Rectangle: "Validated Output"
    Arrow from Provenance Record.

## Visual Style
- Clean, sparse, professional
- The feedback arc is the distinguishing visual feature — make it prominent
- Main pipeline boxes: white (#FFFFFF) fill, dark gray (#333333) outline
- Human Escalation box: light yellow (#FFFBEB) fill
- Flagged Output box: light orange (#FFF3E0) fill — visually distinct from clean pass
- Feedback arc: medium gray (#888888), slightly curved, with arrowhead pointing back to Generator
- Decision diamond: white fill, dark gray outline
- No decorative elements, no gradients, no shadows
- Sans-serif font

## Color Palette
- Background: white or very light warm gray (#F8F7F5)
- Main boxes: white (#FFFFFF), dark gray outline (#333333)
- Human Escalation: light yellow (#FFFBEB)
- Flagged Output: light orange (#FFF3E0)
- Provenance / clean pass: white (#FFFFFF)
- Feedback arc arrow: medium gray (#888888)
- Text: black (#1A1A1A), annotations medium gray (#666666)

## Dimensions
- Landscape orientation, roughly 3:2 aspect ratio
- The feedback loop requires vertical space — slightly taller than fig-05-01

## What This Is NOT
- Not a consensus diagram — there is no comparison between independent models
- No vendor or model names
- The feedback loop is the generator receiving the critic's rejection — not the critic re-running
- The four termination conditions are exits from the loop, not stages in sequence

## Caption
"Generator-critic loop topology. Model A generates; Model B evaluates. Failed outputs return to the generator with specific rejection reasons. Four conditions terminate the loop: all checks pass, confidence threshold met, maximum iterations reached (with flag), or escalation to human review. All paths carry provenance of the iteration trajectory."
